# 植栽建議資料維護說明

> 本檔案是網站「植栽建議工具」的原始資料。一般內容維護只需修改本檔案，不需要修改 TypeScript 程式。

## 維護步驟

1. 修改前先複製一份本檔案備份。
2. 找到要修改的 `## 河川｜河段｜目的` 資料區塊；標題已包含河川、河段與目的，不需要在區塊內重複填寫。
3. 修改清單欄位冒號右側的文字；欄位名稱、半形冒號、行首的 `- ` 及每個區塊的 `##` 不可刪除。
4. 每個欄位獨立一行；同一欄位有多個項目時，使用中文頓號「、」分隔。
5. 圖片只填中文檔名（建議使用「河川-河段-目的.webp」），並將同名 WebP 圖片放入 `public/images/planting-suggestions/`；檔名請勿使用 `/`、`\\`、`:`、`*`、`?`、`"`、`<`、`>`、`|`。
6. 在專案根目錄執行 `npm run dev`，開啟終端機顯示的網址進行確認。

## 新增資料方式

完整複製一個既有的 `##` 資料區塊，貼到檔案最下方後逐欄修改。標題必須維持 `## 河川｜河段｜目的` 格式；`圖片檔名` 與 `卡片標題` 不可留白。每個區塊目前對應一張建議卡片。

> 正式資料由下方第一個符合 `## 河川｜河段｜目的` 格式的區塊開始；上方維護說明不會顯示在網站中。

## 淡水河｜二重疏洪道出口至海口｜固碳（藍碳）
- 棲地類型: 紅樹林、草澤（蘆葦）、沼澤、河岸沙洲、感潮水域
- 溪濱原生植物舉例: 水筆仔、雲林莞草、蘆葦、濱刺麥、白茅、蔓荊、黃槿、苦楝、林投、飄拂草屬、香蒲、野荸薺
- 生態系統服務: 棲地提供、固碳（藍碳）
- 適合環境條件: 植栽需適應鹽害、風害、乾旱。出口河段以濕地、沼澤生態系為主，適合水生或半水生植物；入口河段以草地及河岸沙洲生態系為主，適合潮濕耐淹或生長性強之植物。
- 未來潛在風險: 外來種入侵、河漂垃圾、潮間帶陸域化、溼地陸域化
- 圖片檔名: 淡水河-二重疏洪道出口至海口-固碳藍碳.webp
- 卡片標題: 固碳（藍碳）
- 綠美化建議植栽類型: 建議保留並維持現地灘地植栽，如紅樹林及蘆葦；為避免陸域化加劇，不建議新植。
- 栽植間距: 不建議新植
- 其他管理措施: 可配合工程移除外來種（互花米草等）。
- 備註: 泥灘地、蘆葦草澤及紅樹林為本段重要生態系。工程設計應採最小干擾原則，並保留部分空間作為生態緩衝帶。
